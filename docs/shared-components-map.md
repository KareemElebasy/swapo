# Shared Components Map

Shared components should serve both Buyer and Seller where the interaction model is identical and only data, copy, or actions change.

| Component | Appears in | Why shared | Suggested path | Props | Slots | States | Variants | RTL/LTR and i18n |
|---|---|---|---|---|---|---|---|---|
| `BaseButton` | Auth, home, checkout, add product, modals, landing | Same button anatomy and states across product. | `app/components/base/BaseButton.vue` | `variant`, `size`, `disabled`, `loading`, `icon`, `iconPosition`, `fullWidth`, `to` | default, icon | default, hover, active, disabled, loading | primary, secondary, ghost, danger, link | Icon side must be logical start/end. Text from translation keys. |
| `BaseInput` | Auth, profile address, add product, contact form | Label/input/helper pattern repeats. | `app/components/base/BaseInput.vue` | `modelValue`, `label`, `placeholder`, `required`, `optional`, `error`, `dir`, `type` | prefix, suffix, helper | default, focused, error, disabled | text, phone, number, price, password | Use `dir="auto"` for mixed Arabic/numbers. Required marker position flips with direction. |
| `BaseTextarea` | Product description, notes, contact form | Same field anatomy, taller body. | `app/components/base/BaseTextarea.vue` | `modelValue`, `label`, `placeholder`, `required`, `error`, `maxLength` | helper | default, focused, error, disabled | standard, compact | Arabic copy right aligned, English left aligned. |
| `BaseSelect` | City, district, product status, category, sort | Dropdown fields repeat with arrow icon. | `app/components/base/BaseSelect.vue` | `modelValue`, `options`, `label`, `placeholder`, `searchable`, `error` | option, empty | closed, open, selected, error | menu, popover, sheet | Menus open aligned to logical end in RTL.
